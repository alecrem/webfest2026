# 40分の進行台本

対象: 中学生 (13〜15歳)。ゴール: フロントエンド・バックエンドの雰囲気を
つかみ、自分のページを1つ完成させて誰かに見せる。

事前準備は [README.md](../README.md) の「授業の前に」を参照 (実機
Chromebookでの動作確認、部活・チームデータの確認など)。

---

## 0. 開始前 (授業時間外)

- 生徒に配る1枚のアドレス (https://webfest2026.pages.dev/) を黒板・スライドに用意。
- 教員機で先にページを開き、映せる状態にしておく。

---

## 1. 5分 — フロントエンド / バックエンドってなに？

図 (板書 or スライドでOK):

```
   きみのブラウザ                    バックエンド (このサーバー)
  ┌───────────────┐   アドレスを聞く   ┌───────────────────┐
  │ フロントエンド  │ ───────────────▶ │  データを持っている  │
  │ (見た目・HTML) │ ◀─────────────── │  (ポケモン・部活・   │
  └───────────────┘   データをもらう   │   プロ野球チームなど)│
                                     └───────────────────┘
```

話す内容の例:

- 「ウェブページの見た目を作る部分」= **フロントエンド**。今日はここを
  みんなに触ってもらう。
- 「データを持っていて、聞かれたら答える部分」= **バックエンド**。今日は
  もう用意してあるので、触らなくていい。
- フロントエンドがバックエンドに「このポケモンのデータちょうだい」と
  お願いする、そのお願いのことを **アドレス** と呼ぶ (例:
  `/api/pokemon/25`)。
- 今日使うウィジェット (`<poke-card>` など) は、このお願いを代わりに
  やってくれる部品。

ポイント: 専門用語 (API、URL、JSON) は無理に使わなくてよい。
「アドレス」「データ」「バックエンド」「フロントエンド」の4語で十分。

## 2. 10分 — 全員で1つの変更をやる (デモ)

自分のPC/Chromebookでも、教員の画面を見ながらでもよい。

1. https://webfest2026.pages.dev/ を開く。左にコード、右にプレビュー、下に通信
   パネルがあることを指す。
2. `<poke-card src="/api/pokemon/25" ...>` の `25` を `1` に書きかえる。
3. **💾 保存して表示 (Ctrl+S)** を押す → 右のプレビューが変わることを見せる。
4. 下の「📡 このページの通信」パネルに、いま呼んだアドレスが出ることを
   見せる。「これがさっき話したバックエンドへのお願いだよ」と説明。
5. 色スウォッチ (`color="#ffde00"` の横の四角) をクリックして色を変える →
   もう一度保存して表示。「16進の数字は計算しなくていい、色を選ぶだけ」と
   伝える。
6. 右上の「**バックエンドをのぞく →**」を1回だけ開いて見せる。「ここで
   アドレスを試してから、自分のページに貼ってもいいよ」と伝え、閉じる。

ここまでで [レシピ集](recipes.md) の 1・2 を全員が体験した状態になる。

## 3. 20分 — 自由にカスタマイズ (レシピを見ながら)

- [レシピ集](recipes.md) を1人1部 (印刷 or 画面) 配る。
- 「上から順番にやらなくていい。好きなところから試していい」と伝える。
- 早く終わった生徒には、レシピ集の6 (掛け持ち)・8 (一覧) や
  10・11 (発展) をすすめる。
- 教員は机間巡視。困っている生徒には「🗑️ 最初にもどす」でやり直せることと、
  「バックエンドをのぞく」でアドレスを先に試す方法を思い出させる。
- よくあるつまずき:
  - 保存を押し忘れて「変わらない」→ Ctrl+S か保存ボタンを指す。
  - タグの閉じ忘れなど → エディタの左側の余白 (lintの印) を指す。
  - どのアドレスを使えばいいか分からない → レシピ集の表 (id 一覧) を指す。

---

## 4. 5分 — 発表・共有

- 「見せたい人〜」で数人 (3〜5人目安) に画面を見せてもらう。
- 何を変えたか一言ずつ (色、ポケモン、部活、応援チームなど) を紹介させる。
- 保存はブラウザごと (localStorage) なので、続きをやりたい人には
  **📋 HTMLをコピー** で自分のメモ帳などに保存しておくよう伝える。

---

## 時間が余ったら / 足りなかったら

- 余ったら: レシピ集6〜9をもう1つ試す、他の人のページを見に行く。
- 足りなかったら: 3のカスタマイズ時間を削らず、4の発表人数を減らす
  (全員に発表させない)。
